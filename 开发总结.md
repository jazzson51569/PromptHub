  - `dist_new3/win-unpacked/PromptHub.exe` - x64 架构版本（可直接运行）
  - `dist_new3/win-arm64-unpacked/PromptHub.exe` - arm64 架构版本（可直接运行）
- **新增功能**:
  - 在 Prompt 右键菜单中添加了"折叠全部提示词"功能
  - 位置：在"查看详情"之后
  - 快捷操作：一键折叠所有包含子项的提示词节点

#### 功能实现细节：
- **i18n 翻译**: 在 zh.json 和 en.json 中添加了 `prompt.collapseAllPrompts` 翻译
- **处理函数**: 在 MainContent.tsx 中添加了 `handleCollapseAllPrompts` 函数
  - 逻辑：筛选出所有 `childCount > 0` 的提示词
  - 批量设置这些提示词的 `collapsedPromptIds` 状态
- **UI 集成**: 在右键菜单 `menuItems` 中添加了菜单项
  - 使用 `ChevronDownIcon` 作为图标
  - 点击后自动关闭右键菜单

#### 构建流程：
1. Vite 构建 renderer 进程（约 12 秒）
2. Vite 构建 main 进程（约 2 秒）
3. Vite 构建 preload 进程（约 74 毫秒）
4. electron-builder 打包：
   - x64 unpacked 版本
   - arm64 unpacked 版本
   - 3 个 NSIS 安装包

#### 遇到的问题：
- **文件锁定问题**: 原 dist 目录下的文件被进程锁定，无法删除
  - 原因：可能是之前构建的程序仍在运行或被 IDE 占用
  - 解决方案：修改 electron-builder.json 中的 `output` 配置，使用新目录（如 dist_new3）
  - 临时目录命名建议：`dist_new`、`dist_new2`、`dist_new3` 等

#### 构建警告：
- 部分 chunk 超过 500 kB 限制（特别是 index-HDSewvG_.js 为 503.28 kB）
- 建议优化：考虑使用动态 import() 进行代码分割

#### 注意事项：
- 项目使用 pnpm 作为包管理器，版本为 9.15.0
- Node.js 版本要求: >=24.0.0
- electron-builder 配置文件: `apps/desktop/electron-builder.json`
- 构建配置中 `npmRebuild: false` 和 `nodeGypRebuild: false`，避免重复构建原生模块
- 如果遇到文件锁定问题，使用新目录（如 dist_new2、dist_new3）
- 构建完成后记得恢复 electron-builder.json 中的 output 配置为 "dist"

### 最新经验 (2026-06-28)

#### 任务描述：
- 修复"点击折叠全部提示词后，所有带子节点的一级节点没有折叠"的问题
- 状态作用域问题：右键菜单和处理函数在父组件，而折叠状态在子组件

#### 经验总结：
- **状态提升（State Lifting）**: 当子组件的状态需要被父组件访问或修改时，应将状态提升到共同的祖先组件
- **React 组件通信**: 通过 props 向下传递状态和回调函数，实现父子组件间的状态同步
- **构建输出目录**: 使用 `dist_new10` 作为构建输出目录（累计使用到 dist_new10）

#### 问题分析与解决方案：
- **问题**: `collapsedPromptIds` 状态定义在 `VirtualizedPromptList` 子组件内部，而右键菜单和 `handleCollapseAllPrompts` 函数位于父组件 `MainContent` 中，导致状态更新无法同步到渲染逻辑
- **解决方案**:
  1. 在 `MainContent` 组件中添加 `collapsedPromptIds` 和 `setCollapsedPromptIds` 状态
  2. 添加 `togglePromptCollapse` 和 `handleCollapseAllPrompts` 函数
  3. 修改 `VirtualizedPromptListProps` 接口，添加 `collapsedPromptIds` 和 `onToggleCollapse` props
  4. 修改 `VirtualizedPromptList` 组件，移除内部状态，改为使用传入的 props
  5. 在渲染 `VirtualizedPromptList` 时传入新的 props

#### 关键代码变更：
- **文件**: `apps/desktop/src/renderer/components/layout/MainContent.tsx`
  - 第 716 行：添加 `collapsedPromptIds` 状态
  - 第 1943-1963 行：添加 `togglePromptCollapse` 和 `handleCollapseAllPrompts` 函数
  - 第 352-353 行：修改 `VirtualizedPromptListProps` 接口
  - 第 379-388 行：修改 `VirtualizedPromptList` 组件参数
  - 第 2268-2269 行：传入新的 props

#### 构建产物：
- `dist_new10/win-unpacked/PromptHub.exe` - x64 架构版本
- `dist_new10/win-arm64-unpacked/PromptHub.exe` - arm64 架构版本

#### 功能实现细节：
- **菜单项位置**: 在"移动到..."和"删除"之间
- **显示模式**: 级联菜单，列出所有子节点数量不为零的提示词

#### 技术实现：
- **处理函数**: `handleMoveToPrompt`（第 1811-1826 行）
  - 使用 `movePrompt` 函数将源提示词移动到目标提示词
  - 使用目标提示词的当前子节点数量作为新的 order
- **子菜单项生成**: 在 `menuItems` useMemo 中生成
  - 使用 `getPromptHierarchyMeta` 获取每个提示词的子节点数量
  - 过滤出 `childCount > 0` 且不是自己的提示词
- **图标**: 使用 `GitBranchIcon` 表示节点关系

#### 国际化支持：
- **zh.json**: 添加 `moveToNode`（移动到节点）、`moveToPromptFailed`（移动失败）
- **en.json**: 添加 `moveToNode`（Move to Node）、`moveToPromptFailed`（Failed to move prompt）
