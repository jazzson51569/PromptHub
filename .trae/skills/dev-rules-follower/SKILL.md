# 开发规则遵循者

## 核心规则
    这是一个 eletron 桌面程序
    QT6 SDK 已经安装在 c:\QT 目录下
    cmake 已经安装在 c:\qt 目录下

**这是必须遵循的强制规则，不能有任何例外。**

### 规则1：开发任务开始前
- **必须**读取项目根目录下的 `开发总结.md` 文件
- **必须**仔细阅读并理解其中的经验教训和注意事项
- **必须**在开发过程中遵循这些经验教训
- 如果文件不存在，需要创建该文件

### 规则2：开发任务结束后
- **必须**将本次开发的经验、遇到的问题、解决方案写入 `开发总结.md`
- **必须**记录任何新发现的注意事项、最佳实践或陷阱
- **必须**确保总结内容对后续开发有帮助
- **必须**在每次开发任务结束时，将总结内容写入 `开发总结.md` 文件
- **必须**在每次写入时，添加当前日期作为标题，格式为 `## 最新经验 (YYYY-MM-DD)`
- **必须**在每次写入时，添加当前开发任务的描述，格式为 `### 任务描述：`
- **必须**每次版本完成后，在总结中加入当前版本的完整功能描述
- 总结格式应清晰、有条理

### 规则3：程序测试和调试规范
- **必须**在运行程序测试过程中完整记录运行信息
- **必须**仔细阅读程序运行日志，以便加快调试效率和质量
- **必须**保存测试过程中的关键输出和错误信息
- **必须**分析日志中的异常和警告信息
- 调试时不能只看结果，必须深入分析运行过程

## 触发时机

此skill **必须**在以下情况触发：

1. **开发任务开始时** - 自动触发，读取开发总结
2. **开发任务结束时** - 自动触发，写入开发总结
3. **用户手动调用** - 用户明确要求时

## 执行流程

### 开始阶段
1. 检查 `开发总结.md` 是否存在
2. 如果不存在，创建文件并添加初始说明
3. 读取文件内容
4. 向用户展示关键经验教训
5. 确认理解并开始开发

### 结束阶段
1. 回顾本次开发过程
2. 识别遇到的问题和解决方案
3. 提炼经验教训
4. 将新内容追加到 `开发总结.md`
5. 确保内容格式清晰、易于阅读

## 总结文件格式

建议使用以下格式：

```markdown
# 开发总结

## 最新经验 (2026-02-19)
- 经验1
- 经验2

## 历史经验
### 2026-02-18
- 经验3
- 经验4
```

## 重要提醒

- **严禁跳过**读取或写入开发总结的步骤
- **严禁**在开发过程中忽视已有的经验教训
- **必须**确保每次开发都有价值总结
- 这是强制规则，没有任何例外情况
